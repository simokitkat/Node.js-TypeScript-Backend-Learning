# Lesson 22: Extending `EventEmitter` (Custom Event-Driven Modules)

## 🧠 Mental Model: Inheriting the Event System

Instead of creating a separate `EventEmitter` instance inside your class, your custom class **inherits** from `EventEmitter` itself. This means every instance of your class automatically gains `.emit()` and `.on()` capabilities, allowing it to dispatch and react to its own custom events natively.

## 💻 Minimal Working Example

```js
// pizzashop.js
const EventEmitter = require("node:events");

class PizzaShop extends EventEmitter {
  constructor() {
    super(); // Must call super() to initialize EventEmitter
    this.orderNumber = 0;
  }

  order(size, topping) {
    this.orderNumber++;
    // 'this' now refers to the PizzaShop instance, which is an emitter
    this.emit("order", size, topping);
  }

  displayOrderNumber() {
    console.log(`Current order number: ${this.orderNumber}`);
  }
}

module.exports = PizzaShop;
```

```js
// index.js
const PizzaShop = require("./pizzashop");

const pizzaShop = new PizzaShop();

// Register listener on the instance itself
pizzaShop.on("order", (size, topping) => {
  console.log(`Order received. Making a ${size} pizza with ${topping}`);
});

// Emit event via class method
pizzaShop.order("large", "mushrooms");
pizzaShop.displayOrderNumber();
```

**Terminal Output:**

```
Order received. Making a large pizza with mushrooms
Current order number: 1
```

## 🔑 Core Mechanics

| Concept              | Implementation                        | Why It Matters                                                              |
| -------------------- | ------------------------------------- | --------------------------------------------------------------------------- |
| **Inheritance**      | `class MyModule extends EventEmitter` | Grants access to `.emit()` and `.on()` without a separate emitter object    |
| **Constructor Init** | `super()` inside `constructor()`      | Initializes the underlying `EventEmitter` setup required for event tracking |
| **Emitting**         | `this.emit('eventName', arg1, arg2)`  | `this` refers to the class instance, which now functions as the emitter     |
| **Listening**        | `instance.on('eventName', callback)`  | Listeners attach directly to the class instance from outside                |

## 🛡️ Production-Ready Pattern: Module Decoupling

Events allow different modules to communicate without importing or directly calling each other's methods.

```js
// drinkmachine.js
class DrinkMachine {
  serveDrink(size) {
    if (size === "large") {
      console.log("Serving complementary drink");
    }
  }
}
module.exports = DrinkMachine;

// index.js
const DrinkMachine = require("./drinkmachine");
const drinkMachine = new DrinkMachine();

pizzaShop.on("order", (size, topping) => {
  console.log(`Order received. Making a ${size} pizza with ${topping}`);
  // Decoupled: DrinkMachine reacts to the same event without PizzaShop knowing about it
  drinkMachine.serveDrink(size);
});
```

> ✅ **Benefit**: `PizzaShop` emits an `order` event. `index.js` listens and delegates to `DrinkMachine`. Modules remain independent and reusable.

## 💡 Key Takeaway

Extending `EventEmitter` allows your custom classes to become native event sources. Using `extends EventEmitter` and `super()`, instances inherit `.emit()` and `.on()`, enabling event-driven architecture and loose coupling between modules. **Most core Node.js modules (`fs`, `stream`, `http`) use this exact inheritance pattern.**

## 🧠 Interview Prep Hook

> **Q: "How do you make a custom class event-driven in Node.js?"**  
> **A**: "Import `EventEmitter` from the `events` module and extend your class from it using `extends`. You must call `super()` in the constructor to initialize the event system. Once inherited, use `this.emit()` inside the class to dispatch events and `instance.on()` outside to register listeners."

> **Q: "Why is calling `super()` required when extending `EventEmitter`?"**  
> **A**: "`super()` initializes the parent `EventEmitter` class, setting up internal data structures required for tracking listeners and emitting events. Without it, `this.emit()` will not function correctly."

> **Q: "How does extending `EventEmitter` promote loose coupling?"**  
> **A**: "It allows modules to communicate via events rather than direct method calls. A class emits an event without knowing which modules will listen to it. External modules attach listeners independently, making components reusable, testable, and easier to refactor."
