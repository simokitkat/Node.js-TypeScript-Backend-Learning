# Lesson 20: The Callback Pattern

## 🧠 Mental Model: Functions as Data

In JavaScript, functions are **first-class objects**. This means they can be treated like any other value: passed as arguments to other functions, or returned as results. This capability is the foundation of the callback pattern.

## 📦 Core Terminology

| Term                      | Definition                                                               |
| ------------------------- | ------------------------------------------------------------------------ |
| **Callback function**     | A function passed as an argument to another function                     |
| **Higher-order function** | A function that accepts a function as an argument, or returns a function |

## 💻 Minimal Working Example

```js
// 1. Define a standard function
const greet = (name) => console.log(`hello ${name}`);

// 2. Define a higher-order function that accepts a callback
const higherOrderFunction = (callbackFn) => {
  const name = "Vishwas";
  callbackFn(name); // Executes the passed-in function
};

// 3. Pass the callback function as an argument
higherOrderFunction(greet);
// Output: hello Vishwas
```

## ⏱️ Two Types of Callbacks

### 1. Synchronous Callbacks

- **Behavior**: Executed immediately inside the higher-order function.
- **Example**: The `greet` example above, or array methods like `map`, `filter`, and `sort`.
- **Purpose**: Defines custom logic that the higher-order function applies immediately.

### 2. Asynchronous Callbacks

- **Behavior**: Execution is delayed until a specific time or event occurs.
- **Purpose**: Resume code execution **after** an asynchronous operation completes.
- **Examples**:
  - **Event Handlers**: `element.addEventListener('click', callback)` → runs only when the user clicks.
  - **Data Fetching**: `$.get(url, callback)` (jQuery) → runs only after the network response is received.

## 🌐 Why Callbacks Matter in Node.js

- Most Node.js modules are designed to be **asynchronous** to prevent blocking execution.
- Common async operations: reading files, fetching database data, handling network requests.
- Callbacks allow Node.js to continue processing other tasks while waiting for I/O operations to complete, executing the callback only when the operation finishes.

## 🧠 Interview Prep Hook

> **Q: "What is a callback function in JavaScript?"**  
> **A**: "A callback is a function passed as an argument to another function. The receiving function (a higher-order function) can execute it immediately for synchronous logic, or delay its execution until an asynchronous operation or event completes."

> **Q: "What is the difference between synchronous and asynchronous callbacks?"**  
> **A**: "Synchronous callbacks execute immediately within the higher-order function (e.g., `Array.map`). Asynchronous callbacks delay execution until a future event or operation completes (e.g., DOM events, network requests, file I/O), enabling non-blocking code flow."

> **Q: "Why does Node.js rely heavily on the callback pattern?"**  
> **A**: "Node.js uses callbacks to handle asynchronous operations without blocking the main execution thread. This allows the server to continue handling other requests while waiting for I/O operations like file reads or database queries to finish."

## 💡 Key Takeaway

Callbacks enable delayed execution. Synchronous callbacks apply logic immediately; asynchronous callbacks resume execution after an event or operation completes. This pattern is fundamental to Node.js's non-blocking, event-driven architecture.
