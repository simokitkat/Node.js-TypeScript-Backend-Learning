# Lesson 18: Built-in Modules (Core Modules) — Introduction

## 📦 Module Types Recap

| Type                    | Description                                            | Example              |
| ----------------------- | ------------------------------------------------------ | -------------------- |
| **Local modules**       | Modules you create and share within your application   | `./utils/math.js`    |
| **Built-in modules**    | Modules that Node.js ships with by default             | `path`, `fs`, `http` |
| **Third-party modules** | Modules written by other developers, installed via npm | `express`, `lodash`  |

## 🔧 Built-in Modules (Core Modules)

### Key Facts

- Also referred to as **core modules**
- Available by default when you install Node.js
- **You still need to import the module before you can use it**

### Five Core Modules Covered in This Section

```
1. path    → Path manipulation utilities
2. events  → Event emitter pattern
3. fs      → File system operations
4. stream  → Streaming data handling
5. http    → HTTP server/client functionality
```

### Complexity Note

- **`path` module**: Straightforward to understand with examples
- **`events`, `fs`, `stream`, `http`**: More complex concepts — re-watch videos if needed

### Source Code Reference

- Built-in module source code is located in the **`lib/` folder** of the Node.js repository: https://github.com/nodejs/node
