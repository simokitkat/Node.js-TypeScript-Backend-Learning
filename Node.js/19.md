# Lesson 19: The `path` Module

## 🧠 Mental Model: Path Manipulation Utilities

The `path` module provides built-in utilities for safely working with file and directory paths across different operating systems. It handles platform-specific separators (`/` vs `\`) and path normalization automatically.

## 📦 Import Syntax

```js
// Option 1: Standard (most common)
const path = require("path");

// Option 2: Protocol prefix (recommended)
const path = require("node:path");
```

> ✅ The `node:` prefix is optional. Node.js automatically detects built-in modules, but the prefix is recommended for clarity and future-proofing.

## 🔍 Core Methods

All examples use `__filename` and `__dirname` (absolute path to current file & directory).

### 1. `path.basename(path)`

Returns the last portion of a path.

```js
path.basename(__filename); // 'index.js'
path.basename(__dirname); // 'node.js'
```

### 2. `path.extname(path)`

Returns the extension of the path (including the dot), or empty string if none.

```js
path.extname(__filename); // '.js'
path.extname(__dirname); // '' (empty)
```

### 3. `path.parse(path)`

Returns an object with significant path elements: `root`, `dir`, `base`, `ext`, `name`.

```js
path.parse(__filename);
// → { root: '/', dir: '/users/...', base: 'index.js', ext: '.js', name: 'index' }
```

Access properties via dot notation: `path.parse(__filename).name` → `'index'`

### 4. `path.format(pathObject)`

Reconstructs a path string from an object returned by `parse()`.

```js
path.format(path.parse(__filename)); // Returns original __filename string
```

### 5. `path.isAbsolute(path)`

Returns `true` if path is absolute, `false` if relative.

```js
path.isAbsolute(__filename); // true
path.isAbsolute("./data.json"); // false
```

### 6. `path.join(...paths)`

Joins path segments using the platform-specific separator and **normalizes** the result.

```js
path.join("folder1", "folder2", "index.html");
// → 'folder1/folder2/index.html' (macOS/Linux)

// Normalization examples:
path.join("folder1", "//folder2", "index.html");
// → 'folder1/folder2/index.html' (removes extra slashes)

path.join("folder1", "folder2", "../index.html");
// → 'folder1/index.html' (resolves '..' by moving up one directory)

path.join(__dirname, "data.json");
// → '/full/absolute/path/to/data.json'
```

### 7. `path.resolve(...paths)`

Resolves a sequence of paths into an **absolute path**. Behavior depends on arguments:

```js
path.resolve("folder1", "folder2", "index.html");
// → '/current/working/dir/folder1/folder2/index.html' (prepends cwd if no '/')

path.resolve("/folder1", "folder2", "index.html");
// → '/folder1/folder2/index.html' (treats '/' as root)

path.resolve("folder1", "/folder2", "index.html");
// → '/folder2/index.html' (resets to '/' when encountered, ignores prior segments)

path.resolve(__dirname, "data.json");
// → '/full/path/to/dir/data.json' (__dirname is already absolute)
```

> 💡 `resolve` processes arguments sequentially, updating the root whenever a leading `/` appears, and always returns an absolute path.

## 🌐 Why Use the `node:` Prefix?

The `node:` protocol (`require('node:path')`) is optional but recommended because it:

1. Clearly indicates the import is a Node.js built-in module
2. Makes the import identifier a valid absolute URL
3. Avoids naming conflicts with future Node.js built-in modules

## 💡 Key Takeaway

The `path` module handles cross-platform path resolution, normalization, and parsing. Use `join()` to safely concatenate paths, and `resolve()` when you need guaranteed absolute paths. Always prefer `path.join()` over string concatenation to avoid OS-specific separator bugs.

## 🧠 Interview Prep Hook

> **Q: "What's the difference between `path.join()` and `path.resolve()`?"**  
> **A**: "`path.join()` concatenates path segments using the platform separator and normalizes the result, but doesn't guarantee an absolute path. `path.resolve()` processes segments sequentially into an absolute path, using the current working directory as a fallback root if no `/` is provided, and resets to a new root when a `/` appears in the sequence."

> **Q: "Why should you use `require('node:path')` instead of `require('path')`?"**  
> **A**: "The `node:` prefix explicitly identifies the import as a built-in Node.js module, makes the identifier a valid absolute URL, and prevents future naming conflicts if a third-party package or future language feature introduces a similarly named module."

> **Q: "How does `path.join()` handle `../` in path segments?"**  
> **A**: "`path.join()` normalizes the path by resolving `..` segments to move up one directory level. For example, `path.join('a', 'b', '../c')` resolves to `a/c`, removing the redundant directory traversal."
