# Lesson 21: The `events` Module

## 🧠 Mental Model: The Restaurant Order System

Think of events like a restaurant ticket system:

- **Placing an order** = Emitting an event (`emit`)
- **Kitchen seeing the ticket** = Listener registration (`on`)
- **Cooking the pizza** = Callback execution
- **Multiple cooks** = Multiple listeners for the same event
  The system doesn't stop to wait for the pizza to finish; it just signals when something happens and lets registered handlers respond.

## 📦 Import & Setup

```js
const EventEmitter = require("node:events");
const emitter = new EventEmitter();
```

> ✅ The `events` module exports the `EventEmitter` class directly. You must instantiate it to work with events.

## 🔧 Core API: `.on()` and `.emit()`

| Method                         | Purpose                                                     | Syntax                                   |
| ------------------------------ | ----------------------------------------------------------- | ---------------------------------------- |
| `emitter.on(event, listener)`  | Register a callback that runs when the event is emitted     | `emitter.on('event name', callbackFn)`   |
| `emitter.emit(event, ...args)` | Broadcast/dispatch an event to trigger registered listeners | `emitter.emit('event name', arg1, arg2)` |

### 💻 Minimal Working Example

```js
// 1. Register listener (delayed execution via callback)
emitter.on("order pizza", (size, topping) => {
  console.log(`Baking a ${size} pizza with ${topping}`);
});

// 2. Emit event (triggers the listener)
emitter.emit("order pizza", "large", "mushrooms");
// Output: Baking a large pizza with mushrooms
```

## 🔑 Key Behaviors

### 1. Passing Data to Listeners

Arguments passed after the event name in `.emit()` are automatically forwarded to the listener callback as parameters.

```js
emitter.emit("order pizza", "large", "mushrooms");
// 'large' → size parameter
// 'mushrooms' → topping parameter
```

### 2. Multiple Listeners per Event

You can register multiple callbacks for the same event. They all execute in the order they were registered.

```js
// First listener
emitter.on("order pizza", (size) => {
  console.log(`Order received. Baking pizza.`);
});

// Second listener for the same event
emitter.on("order pizza", (size) => {
  if (size === "large") {
    console.log("Serving complementary drink");
  }
});

emitter.emit("order pizza", "large");
// Output:
// Order received. Baking pizza.
// Serving complementary drink
```

### 3. Non-Blocking Execution

Registering listeners and emitting events does **not** block the main execution thread.

```js
console.log("Do work before event occurs in the system");

emitter.emit("order pizza", "large", "mushrooms");

console.log("Code continues without waiting");
```

> ✅ Execution flows linearly. Listeners are delayed until the event is explicitly signaled. This is the foundation of **event-driven programming** in Node.js.

## 💡 Key Takeaway

The `events` module enables custom, non-blocking event handling in Node.js. Using `EventEmitter`, you register listeners with `.on()` and trigger them with `.emit()`. Arguments pass seamlessly between emitter and listeners, and multiple listeners can respond to the same event without blocking execution.

## 🧠 Interview Prep Hook

> **Q: "How does the `events` module work in Node.js?"**  
> **A**: "The module provides the `EventEmitter` class. You instantiate it, register callbacks with `.on(eventName, listener)`, and trigger them with `.emit(eventName, ...args)`. When an event is emitted, Node.js automatically executes all registered listeners for that event, passing along any provided arguments."

> **Q: "Why is the `events` module considered non-blocking?"**  
> **A**: "Registering listeners just stores callbacks for later execution. When `.emit()` is called, the event is dispatched without pausing the main thread. The listeners execute asynchronously as part of Node's event-driven model, allowing the application to continue processing other tasks."

> **Q: "Can multiple functions listen to the same event?"**  
> **A**: "Yes. You can register multiple listeners for the same event using `.on()`. Each listener is executed in the order it was registered when the event is emitted."
