# Lesson 23: Character Sets and Encoding

## 🧠 Mental Model: The Translation Chain

```
Character ('V')
   ↓
Character Set (Unicode) → Maps to number (86)
   ↓
Character Encoding (UTF-8) → Maps to binary (01010110)
   ↓
Stored in computer as bits
```

## 🔢 Binary Data Basics

### What Is Binary?

- Computers store and represent data in **binary format**: a collection of zeros and ones.
- Each `0` or `1` is called a **binary digit** or **bit** for short.

### Converting Numbers to Binary

To work with data, a computer converts it to its binary representation using the **base-2 numeric system**.

**Example: Number 4 → Binary**

```
Binary: 1 0 0

Calculation:
(2^0 × 0) + (2^1 × 0) + (2^2 × 1)
= 0 + 0 + 4
= 4
```

## 🔤 Character Sets

### The Problem

Numbers are straightforward to convert to binary. But how does a computer represent a **character** like `'V'`?

### The Solution: Two-Step Conversion

1. Convert the character to a **number** (character code)
2. Convert that number to **binary**

**Example: Character 'V'**

```js
// In browser console
"V".charCodeAt(0); // Returns: 86
```

- `86` is the **numeric representation** (character code) of `'V'`.

### What Is a Character Set?

> **Character sets** are predefined lists of characters represented by numbers.

| Character Set | Description                       |
| ------------- | --------------------------------- |
| **Unicode**   | Modern standard; `'V'` = `86`     |
| **ASCII**     | Older standard; subset of Unicode |

**Verification**: Visit [unicodeable.com](https://unicodeable.com) → Click uppercase `V` → Shows `86`.

## 🔐 Character Encoding

### What Is Character Encoding?

> **Character encoding** dictates how to represent a number (from a character set) as **binary data** before it can be stored in a computer.

Specifically, it defines **how many bits to use** to represent a number.

### UTF-8 Encoding (Example)

- **UTF-8** states that characters should be encoded in **bytes**.
- **1 byte = 8 bits** (eight ones or zeros).

#### Encoding Examples

| Value           | Decimal | Binary (raw) | UTF-8 (8-bit byte) |
| --------------- | ------- | ------------ | ------------------ |
| Number `4`      | 4       | `100`        | `00000100`         |
| Character `'V'` | 86      | `1010110`    | `01010110`         |

> 💡 UTF-8 pads shorter binary values with leading zeros to form complete bytes.

## 💡 Key Takeaway

Computers store characters by:

1. Using a **character set** (e.g., Unicode) to map characters to numbers.
2. Using a **character encoding** (e.g., UTF-8) to map those numbers to fixed-length binary (bytes).
3. Storing the resulting bits in memory or on disk.

This same principle applies to other data types like images and videos, which also follow encoding guidelines for binary storage.

## 🧠 Interview Prep Hook

> **Q: "What is the difference between a character set and character encoding?"**  
> **A**: "A character set (like Unicode) defines which number represents each character. Character encoding (like UTF-8) defines how those numbers are converted into binary bits for storage. The set is the mapping; the encoding is the representation format."

> **Q: "How does a computer store the string 'V' in memory?"**  
> **A**: "First, the character set (Unicode) maps 'V' to the number 86. Then, the encoding (UTF-8) converts 86 to an 8-bit binary byte: 01010110. This byte is what gets stored in memory."

> **Q: "What is a byte, and why does UTF-8 use it?"**  
> **A**: "A byte is a group of 8 bits. UTF-8 uses bytes as the unit for encoding characters, ensuring consistent storage size and compatibility across systems. For ASCII-range characters like 'V', UTF-8 uses exactly one byte."
